# 设计规范

## 1. 关键状态覆盖

每个页面/组件设计必须覆盖以下状态：

| 状态 | 设计要求 |
|------|----------|
| Loading | 骨架屏/加载动画，避免白屏等待 |
| Empty | 空数据插画 + 引导文案 + 操作入口 |
| Error | 错误提示 + 重试/返回操作 |
| Permission | 无权限时的提示与申请/返回引导 |
| Offline | 离线提示，缓存数据降级展示 |

## 2. 用户流程设计

### 设计原则
- 以"用户如何完成任务"为核心，而非"页面长什么样"
- 每个流程不超过 5 步完成核心任务
- 关键路径上的操作一键可达
- 减少用户记忆负担，信息在需要时呈现

### 流程文档
- 使用流程图/序列图描述关键用户流程
- 标注每个步骤的前置条件与后置结果
- 明确分支流程与异常路径

## 3. 交互规范

### 反馈原则
- 操作可撤销的提供撤销入口
- 不可逆操作（删除、支付）提供二次确认
- 表单校验实时反馈，而非提交后才提示
- 长时间操作显示进度，支持取消

### 导航
- 用户始终知道自己在哪、能去哪、怎么回去
- 面包屑导航层级不超过 4 层
- 移动端优先考虑底部 Tab 导航

## 4. 多端适配

- 默认以移动端优先设计，再适配桌面端
- 断点参考：320px（小手机）、375px（手机）、768px（平板）、1024px（小桌面）、1280px+（桌面）
- 触控区域不小于 44x44px
- 文本行宽不超过 80 字符以保证可读性

## 5. 可访问性

- 颜色对比度满足 WCAG AA 标准（正文 4.5:1，大文本 3:1）
- 不仅依赖颜色传达信息，配合图标/文字
- 图片提供 alt 文本
- 表单输入框关联 label
- 支持键盘导航，焦点状态可见

## 6. 一致性

- 相同功能使用相同的交互模式
- 相同含义使用相同的文案
- 按钮、输入框、卡片等组件样式统一
- 间距使用 4px/8px 倍数体系
- 颜色使用设计系统定义的色板，不随意新增

## 7. 设计交付

- 输出包含交互说明，不只是视觉稿
- 标注组件状态变化与动画过渡
- 提供设计稿链接与切图资源
- 复杂交互提供可交互原型
