# 测试规范

## 1. 测试分层

| 层级 | 范围 | 占比 | 执行速度 | 维护成本 |
|------|------|------|----------|----------|
| 单元测试 | 函数/方法/类 | 70% | 快（ms） | 低 |
| 集成测试 | 模块间交互/API | 20% | 中（s） | 中 |
| E2E 测试 | 完整用户流程 | 10% | 慢（min） | 高 |

## 2. 单元测试规范

### 测试原则
- 每个公开函数/方法必须有对应测试
- 测试覆盖：正常输入、边界值、异常输入、空值/null
- 测试独立，不依赖其他测试的执行顺序
- 不依赖外部服务（数据库、网络、文件系统），使用 Mock
- 测试命名：`{被测方法}_{场景}_{期望结果}`

### 前端单元测试
```typescript
// 组件测试示例
describe('UserProfile', () => {
  it('正常渲染用户信息', () => { /* ... */ });
  it('loading 状态显示骨架屏', () => { /* ... */ });
  it('error 状态显示错误提示和重试按钮', () => { /* ... */ });
  it('用户名为空时显示默认名称', () => { /* ... */ });
  it('点击编辑按钮触发 onEdit 回调', () => { /* ... */ });
});
```

### 后端单元测试
```python
# 服务层测试示例
class TestOrderService:
    def test_create_order_success(self):
        """正常创建订单"""
        pass

    def test_create_order_insufficient_stock(self):
        """库存不足时抛出异常"""
        pass

    def test_create_order_invalid_product(self):
        """商品不存在时抛出异常"""
        pass

    def test_create_order_zero_quantity(self):
        """数量为 0 时抛出参数异常"""
        pass
```

## 3. 集成测试规范

### API 集成测试
- 每个 API 接口至少覆盖：正常请求、参数校验失败、未认证、无权限
- 测试数据库使用独立测试库，每次测试前重置数据
- 关键业务流程（注册→登录→操作→登出）端到端验证

### 测试用例模板
```
接口：POST /api/orders
测试用例：
1. 正常创建订单 → 201，返回订单详情
2. 缺少必填字段 → 400，返回字段校验错误
3. 未携带 Token → 401
4. 无下单权限 → 403
5. 商品不存在 → 404
6. 库存不足 → 409
7. 并发下单 → 库存扣减正确，不超卖
```

## 4. E2E 测试规范

### 覆盖范围
- 核心业务流程（用户注册、登录、核心功能操作）
- 跨页面流程（列表→详情→编辑→保存→返回列表）
- 权限验证（不同角色看到不同内容）

### 编写原则
- 只覆盖核心 Happy Path，不追求覆盖率
- 使用 data-testid 定位元素，不依赖 CSS 类名
- 测试数据使用固定测试账号，与生产数据隔离
- 失败时截图保存，便于排查

## 5. 测试数据管理

- 测试数据与生产数据严格隔离
- 使用工厂函数/Fixture 生成测试数据，不硬编码
- 每个测试用例创建独立数据，测试后清理
- 不使用生产数据脱敏后测试（仍有泄露风险）

## 6. 测试覆盖率

| 指标 | 最低要求 | 推荐目标 |
|------|----------|----------|
| 行覆盖率 | 70% | 80% |
| 分支覆盖率 | 60% | 75% |
| 函数覆盖率 | 75% | 85% |

- 核心业务逻辑覆盖率不低于 85%
- 覆盖率不达标不允许合并代码
- 不追求 100% 覆盖率，关注测试质量而非数量

## 7. Mock 与 Stub 规范

- 只 Mock 外部依赖（数据库、HTTP、文件系统、第三方服务）
- 不 Mock 被测模块内部的纯函数
- Mock 数据尽量接近真实数据结构和规模
- 验证 Mock 的调用次数和参数，不只是让测试通过

## 8. 测试命名规范

```
{被测对象}_{测试场景}_{期望行为}
```

### 示例
```
createOrder_validInput_returnsOrderDetail
createOrder_emptyUserId_throwsValidationError
createOrder_negativeQuantity_throwsBusinessError
UserListPage_loading_showsSkeleton
UserListPage_empty_showsEmptyState
UserListPage_error_showsRetryButton
```

## 9. CI/CD 中的测试

- 提交代码时自动运行单元测试（< 2 分钟）
- 合并请求时运行单元测试 + 集成测试（< 10 分钟）
- 发布前运行完整测试套件（含 E2E）
- 测试失败阻止合并和部署
- 测试报告自动生成并归档
