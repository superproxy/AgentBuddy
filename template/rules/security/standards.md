# 安全规范

## 1. 认证与授权

### 认证
- 所有非公开接口必须验证用户身份
- Token 通过 `Authorization: Bearer <token>` 传递
- Access Token 有效期不超过 2 小时，Refresh Token 不超过 7 天
- Token 存储在 HttpOnly Cookie 或安全的内存存储中，不存储在 localStorage
- 登录失败超过 5 次锁定账号 15 分钟
- 密码强度要求：至少 8 位，包含大小写字母、数字、特殊字符中至少 3 种

### 授权
- 默认拒绝：未明确授权的操作一律禁止
- 最小权限原则：用户只拥有完成任务所需的最小权限
- 权限校验在服务端进行，前端权限控制仅用于 UI 展示
- 敏感操作（删除、权限变更、支付）需二次验证

## 2. 数据安全

### 传输安全
- 生产环境强制 HTTPS，禁用 HTTP
- API 响应头设置安全头：
  - `Strict-Transport-Security: max-age=31536000`
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: DENY`
  - `X-XSS-Protection: 1; mode=block`

### 存储安全
- 密码使用 bcrypt/argon2 哈希存储，禁止明文或 MD5
- 敏感数据（手机号、身份证、银行卡）加密存储
- 密钥通过密钥管理服务（KMS）或环境变量管理，不硬编码
- 数据库连接字符串、API Key 等凭据不提交到代码仓库

### 脱敏规则
| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| 手机号 | 中间 4 位替换为 **** | 138****1234 |
| 身份证 | 中间 10 位替换为 **** | 310***********1234 |
| 银行卡 | 仅显示后 4 位 | ****1234 |
| 邮箱 | 用户名部分脱敏 | u***@example.com |
| 姓名 | 保留首尾字符 | 张*三 |

## 3. 输入校验

### 通用原则
- 所有用户输入必须在服务端校验，前端校验仅用于用户体验
- 白名单校验优于黑名单校验
- 校验规则：类型、长度、格式、范围、业务逻辑

### 防注入
- SQL：使用参数化查询或 ORM，禁止拼接 SQL
- XSS：用户输入输出到页面时做 HTML 转义
- 命令注入：不直接调用系统命令，使用安全的 API
- 路径遍历：校验文件路径，限制在允许的目录内

### 文件上传
- 校验文件类型（白名单），不只依赖扩展名
- 限制文件大小（默认最大 10MB）
- 文件重命名为随机字符串，保留扩展名
- 上传目录设置不可执行权限
- 图片文件做二次渲染，去除恶意代码

## 4. 接口安全

### 频率限制
- 登录接口：同一 IP 每分钟不超过 10 次
- 验证码发送：同一手机号每分钟不超过 1 次，每天不超过 10 次
- 普通 API：同一用户每分钟不超过 60 次
- 超限返回 429 Too Many Requests

### 请求防护
- 关键操作使用 CSRF Token
- API 设置合理的请求体大小限制（默认 1MB）
- 接口响应不暴露服务器信息（框架版本、错误堆栈）
- 生产环境关闭调试模式和详细错误信息

## 5. 日志与审计

### 审计日志
以下操作必须记录审计日志：

| 操作类型 | 记录内容 |
|----------|----------|
| 登录/登出 | 用户 ID、IP、时间、结果 |
| 权限变更 | 操作人、被操作人、变更内容、时间 |
| 敏感数据访问 | 访问人、数据类型、时间 |
| 删除操作 | 操作人、删除对象、时间 |
| 支付/交易 | 交易双方、金额、时间、结果 |
| 配置变更 | 变更人、变更内容、时间 |

### 日志安全
- 日志中不记录密码、Token、完整身份证号
- 敏感字段脱敏后记录（如手机号 138****1234）
- 日志保留至少 90 天，审计日志保留至少 1 年
- 日志访问需授权，防止未授权查看

## 6. 依赖安全

- 定期扫描第三方依赖漏洞（使用 npm audit / pip audit / OWASP Dependency-Check）
- 及时更新有安全漏洞的依赖
- 不使用来源不明或已停止维护的第三方库
- 锁定依赖版本，避免自动升级引入风险

## 7. 安全编码检查清单

- [ ] 所有用户输入是否经过服务端校验
- [ ] 是否使用参数化查询防止 SQL 注入
- [ ] 用户输出到页面是否做了 XSS 防护
- [ ] 密码是否使用强哈希算法存储
- [ ] 敏感数据是否加密存储和脱敏展示
- [ ] 接口是否有频率限制
- [ ] 关键操作是否有审计日志
- [ ] 生产环境是否关闭调试信息
- [ ] 第三方依赖是否有已知漏洞
- [ ] 文件上传是否有类型和大小限制
